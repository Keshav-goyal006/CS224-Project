How to call memory-map/Makefile from mem_generator/Makefile
===========================================================

Do this in:

  D:/New/CS224/CS224-Project/mem_generator/Makefile

Do not replace the whole file. Add these variables near the top:

  PATCH_BIT ?= 0
  MEMORY_MAP_DIR ?= memory-map
  BIT_IN ?= D:/New/Image_processing/Image_processing.runs/impl_1/top_fpga.bit
  BIT_OUT ?= D:/New/Image_processing/Image_processing.runs/impl_1/fast_update.bit


Then, inside the existing `vision` recipe, after this line:

  cd imem_dmem && python bin2hex.py

add this block:

  if [ "$(PATCH_BIT)" = "1" ]; then \
      $(MAKE) -C $(MEMORY_MAP_DIR) flash_fast BIT_IN="$(BIT_IN)" BIT_OUT="$(BIT_OUT)"; \
  fi


How to run
==========

Normal behavior, only compile and generate memory files:

  make vision

Compile, generate memory files, then call memory-map/Makefile:

  make vision PATCH_BIT=1

With a different bitstream path:

  make vision PATCH_BIT=1 BIT_IN="D:/path/to/top_fpga.bit" BIT_OUT="D:/path/to/fast_update.bit"


What this does
==============

The parent `make vision` still builds `vision.elf`, `imem.bin`, `dmem.bin`, and
then runs `bin2hex.py`.

Only when `PATCH_BIT=1`, it also runs:

  make -C memory-map flash_fast BIT_IN="..." BIT_OUT="..."

That second command enters:

  D:/New/CS224/CS224-Project/mem_generator/memory-map

and calls the `flash_fast` target in that Makefile.


Important
=========

Since your memory-map Makefile currently uses:

  IMEM_HEX ?= ../imem_dmem/imem.mem

make sure `make vision` really generates:

  D:/New/CS224/CS224-Project/mem_generator/imem_dmem/imem.mem

before the `$(MAKE) -C memory-map flash_fast ...` step runs.

If your generated file is named `imem.hex` instead, either update the
memory-map Makefile to use `../imem_dmem/imem.hex` only if your Vivado
`updatemem` accepts that format, or generate a real `imem.mem` file from
`bin2hex.py`.
